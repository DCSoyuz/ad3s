# ad3s_prog_esp32

Прошивка **ESP32-S3** (ESP-IDF, C++) — USB↔SPI мост между настольным
приложением **workstation4ad3s** (Java) и микросхемой преобразователя
угол→код **5400ТР065А-022** (АО «Дизайн Центр «Союз»»).

ESP32 и workstation4ad3s работают в паре: приложение подключается к
контроллеру по USB (виртуальный COM-порт) и через него читает/пишет регистры
микросхемы. Скачать приложение: <https://ftp.dcsoyuz.ru/soft/workstation4ad3s/>.

Назначение проекта — дать инженерам пример того, как ESP32 взаимодействует с
указанной микросхемой: как формируются пакеты, как читаются/пишутся регистры
по SPI, как работают DMA-режимы и энкодеры.

---

## Архитектура

```
┌─────────────────────┐   USB CDC    ┌───────────────┐   SPI   ┌─────────────────────┐
│ workstation4ad3s    │ ◄──────────► │  ESP32-S3     │ ◄─────► │  5400ТР065А-022     │
│ (Java,FlatLaf)      │  виртуальный │  ad3s_prog_   │  10 МГц │  (RDC, ДЦ Союз)     │
│                     │    COM-порт  │  esp32        │         │                     │
└─────────────────────┘              └───────────────┘         └─────────────────────┘
                                            │
                                            ├── GPIO: NRESET, STNDBY, VPP9V, SAMPLE, VC, NSEN
                                            └── PCNT: 2 квадратурных энкодера (ENC1/ENC2)
```

ESP32 выступает ведомым по USB-CDC (TinyUSB): принимает бинарные пакеты от
Java-приложения, разбирает их, выполняет SPI-транзакции к микросхеме и
возвращает ответ.

## Формат пакета

Пакет — бинарный, little-endian, последний байт — контрольная сумма
(сумма всех предыдущих байт по модулю 256).

**Запрос** (поля `A_REQ_*`):

| offset | поле          | описание                                  |
|--------|---------------|-------------------------------------------|
| 0      | STARTBYTE     | стартовый байт                            |
| 1..2   | NUMB L/H      | общая длина пакета (байт)                 |
| 3      | COMMAND_WORD  | код команды                               |
| 4..5   | ADDRESS L/H   | адрес (16-битное слово)                   |
| 6..7   | LEN L/H       | число 16-битных слов данных              |
| 8..    | VALUE0 L/H... | данные (по 2 байта на слово)              |
| N-1    | CHECKSUM      | контрольная сумма                         |

**Ответ** (`A_RESP_*`): `0x55` STARTBYTE, NUMB L/H, COMMAND_WORD,
ADDRESS L/H, NUMW L/H, данные, CHECKSUM.

## Команды

| код   | имя                       | действие                                            |
|-------|---------------------------|-----------------------------------------------------|
| 0x01  | READ                      | чтение `LEN` слов с `ADDRESS`                       |
| 0x02  | WRITE                     | запись `LEN` слов по `ADDRESS`                      |
| 0x05  | RANDOM_READ               | чтение набора адресов из payload                    |
| 0x06  | WRITE_RANDOM              | запись по набору адресов                            |
| 0x0E  | READ_HANDTAP              | чтение HandTap-данных (384 слова)                   |
| 0x10  | SET_NRESET                | управление линией сброса микросхемы                 |
| 0x0F  | SET_STNDBY                | standby                                             |
| 0x15  | SET_VPP9V                 | программное напряжение VPP                          |
| 0x12  | SET_VC / 0x13 SET_SDI     | режимы DMA Quad                                     |
| 0x11  | SET_DMA_QUAD              | одно-/многоканальный DMA-захват (Quad SPI)          |
| 0x14  | SET_DMA_SINGLE            | одиночный DMA-захват                                |
| 0x17  | READ_CPU_REGS             | чтение регистров CPU0/CPU2 микросхемы               |
| 0x18  | SET_SPI_SPEED             | изменение частоты SPI (кГц)                         |
| 0x19/0x1A | ENC_ON / ENC_OFF       | включение/выключение энкодеров (PCNT)               |
| 0x1B  | READ_ENCODERS             | чтение счётчиков ENC1/ENC2                          |
| 0x1C/0x1D | START_RECORD / STOP_RECORD | циклическая запись набора адресов               |
| 0x22  | SET_ELEC_OFFSET           | электронный сдвиг                                   |
| 0x16  | WRITE_BOTP                | запись OTP c управлением VPP                        |

Полный список констант — в `main/ad3s_types.hpp` (`COMMAND_*`), разбор — в
`main/command_handler.hpp` (`CommandHandler::main_task`).

## Структура исходников (`main/`)

| файл                         | роль                                                          |
|------------------------------|---------------------------------------------------------------|
| `main.cpp`                   | `app_main`: инициализация NVS, SPI, USB-CDC, задач            |
| `usb_cdc.hpp/.cpp`           | транспорт USB-CDC (TinyUSB), приём/отправка пакетов           |
| `command_handler.hpp`        | разбор пакетов, диспетчер команд, формирование ответов        |
| `spi_maker.hpp/.cpp`         | SPI read/write слов в микросхему (фрейминг с чётностью)       |
| `ad3s_types.hpp`             | константы, карты пинов, коды команд, смещения полей пакета    |
| `esp32_dma_spi_master.hpp`   | vendored DMA SPI Master (arduino-esp32-dma-spi, MIT)          |
| `encoder_pcnt.hpp`           | драйвер квадратурных энкодеров на PCNT                        |
| `led_util.hpp/.cpp`          | статусный светодиод (WS2812 / GPIO)                           |
| `console_handler.hpp/.cpp`   | отладочная консоль по UART (команды `led on/off`, `blink`)    |

## Сборка и прошивка

Ниже — пошаговая инструкция от установки среды до заливки прошивки в
контроллер. Минимум для запуска (кратко): ESP-IDF v5.5.2 или свежее 5.5.x,
таргет `esp32s3`:

```cmd
idf.py set-target esp32s3
idf.py menuconfig     ;; по умолчанию «Dev board for 065A-022»;
                      ;; для своего модуля выберите «Custom User Board»
idf.py build
idf.py -p COM3 flash monitor
```

### Что понадобится

- **Плата ESP32-S3** — любая с выводом USB: ESP32-S3-DevKitC-1, модуль
  ESP32-S3-WROOM-1 на плате-переходнике и т.п. Важно: именно **S3** (на обычном
  ESP32 прошивка не запустится). Для работы с микросхемой — плата с
  5400ТР065А-022 либо отдельный модуль, подключённый проводами.
- **USB-кабель с передачей данных** (не «зарядный»). Очень частая причина
  «не прошивается» — кабель без данных.
- **ПК** под управлением Windows 10/11. На macOS/Linux действия аналогичны, но
  команды ниже приведены для Windows.
- Доступ к корпоративному FTP для скачивания workstation4ad3s (ссылка вверху).

### Установка VS Code

1. Скачайте установщик: <https://code.visualstudio.com/Download>
   (сборка **Windows x64 User Installer**).
2. Запустите и пройдите мастер — значения по умолчанию подходят.
3. Запустите VS Code.

### Установка ESP-IDF

Проект оттестирован на **ESP-IDF v5.5.2**. Подойдёт и любая более свежая версия
серии **5.5.x** (5.5.3, 5.5.4 и т.д.) — за выходом минорных версий ESP-IDF не
угнаться, но критичных для проекта отличий между ними нет. Если на сайте
доступна версия новее 5.5.2 — смело ставьте её.

ESP-IDF устанавливается офлайн-установщиком. Расширение для VS Code можно
поставить дополнительно — оно само найдёт уже установленный фреймворк и добавит
кнопки сборки и прошивки в редакторе. Дальше собирать и прошивать проект можно из
командной строки (`idf.py`) или из VS Code — в каждом шаге ниже показаны оба
варианта.

**Фреймворк ESP-IDF — офлайн-установщик.**

1. Скачайте **ESP-IDF Tools Installer**: <https://dl.espressif.com/dl/esp-idf/>.
2. При установке выберите версию **5.5.2** (или более свежую `5.5.x`, если есть
   на сайте). Установщик развернёт фреймворк, тулчейн `xtensa-esp32s3-elf`, CMake,
   Ninja, Python и создаст ярлык **«ESP-IDF 5.5.2 CMD»**.
3. Команды `idf.py` из инструкций ниже выполняются в окне **«ESP-IDF 5.5.2 CMD»**.

> Фреймворк ставится отдельно от расширения: в расширении v2.x убран старый мастер
> `Configure ESP-IDF Extension`, поэтому сначала идёт офлайн-установщик, а
> расширение лишь подключается к готовому фреймворку.

**Расширение для VS Code.**

1. В VS Code: `Ctrl+Shift+X` → найдите **ESP-IDF** (издатель *Espressif Systems*)
   → **Install**.
2. Версия расширения (сейчас v2.x) — это **не** версия фреймворка ESP-IDF;
   фреймворк 5.5.x уже установлен на предыдущем шаге.
3. Расширение само найдёт его (читает манифест EIM `eim_idf.json`). Если версия не
   подхватилась — `F1` → **ESP-IDF: Select Current ESP-IDF Version** → выберите
   `5.5.x`.
4. Проверьте связку: `F1` → **ESP-IDF: Doctor Command** — должно показать путь к
   ESP-IDF 5.5.x без ошибок.

Документация Espressif:
<https://docs.espressif.com/projects/esp-idf/en/latest/esp32s3/get-started/>

### Открытие проекта

- **В VS Code:** `File → Open Folder…` → выберите каталог `ad3s_prog_esp32`.
- **В командной строке ESP-IDF:**
  ```cmd
  cd <путь>\ad3s_prog_esp32
  ```

### Настройка платы и пинов

**Выбор таргета и варианта платы.** В терминале ESP-IDF:

```cmd
idf.py set-target esp32s3
idf.py menuconfig
```

В меню **AD3S Board Configuration → Version board** выберите вариант платы:

- **Dev board for 065A-022** — заводская отладочная плата с микросхемой
  5400ТР065А-022 (выбрана по умолчанию);
- **Custom User Board** — любой другой модуль ESP32-S3, подключённый к
  микросхеме проводами; пины задаются вручную (см. ниже).

В VS Code вместо `idf.py menuconfig` — `F1` → **ESP-IDF: SDK Configuration Editor**.

**Статусный светодиод.** В меню **AD3S Board Configuration → Blink LED type**:

- обычный светодиод → **GPIO** + его номер в **Blink GPIO number** (часто `2`, `48`);
- адресная лента WS2812 → **LED strip** + пин ленты;
- если светодиода нет / не уверены → **GPIO** и номер любого свободного пина
  (безопасный вариант: иначе инициализация ленты на неверном пине может вызвать
  `panic` и постоянную перезагрузку).

**Своя плата (Custom User Board) — назначение пинов.** Сначала в `menuconfig`
выберите **Custom User Board**, затем откройте `main/ad3s_types.hpp`, найдите
первый блок `#if defined(CONFIG_CUSTOM_USER_BOARD)` (помечен как `CUSTOM USER BOARD`)
и замените номера GPIO на свои. Значения по умолчанию — рабочая разводка образца;
поменяйте те, что не совпадают с вашей. Подключение к микросхеме **5400ТР065А-022**:

| Константа в `ad3s_types.hpp` | Сигнал                       | Куда подключать                    |
|------------------------------|------------------------------|------------------------------------|
| `GPIO_MOSI`                  | SPI MOSI (ESP → чип)         | вход данных чипа (SDI)             |
| `GPIO_MISO`                  | SPI MISO (чип → ESP)         | выход данных чипа (SDO)            |
| `GPIO_SCLK`                  | SPI тактовый                 | SCLK чипа                          |
| `GPIO_CS`                    | SPI Chip Select              | SSTR чипа (строб кадра/CS)         |
| `GPIO_NRESET`                | сброс                        | nRESET чипа                        |
| `STNDBY_PIN`                 | standby                      | STANDBY                            |
| `VPP9V_PIN`                  | управление VPP               | вход управления VPP                |
| `GPIO_SAMPLE`                | sample                       | SAMPLE                             |
| `GPIO_VC`                    | VC                           | VC                                 |
| `GPIO_NSEN`                  | nSEN                         | nSEN                               |
| `GPIO_ENC1_0/_A/_B`          | энкодер 1 (index/A/B)        | опционально (для команд ENC_*)     |
| `GPIO_ENC2_0/_A/_B`          | энкодер 2 (index/A/B)        | опционально (для команд ENC_*)     |

Минимально для старта нужны 4 линии SPI (`MOSI`, `MISO`, `SCLK`, `CS`) и
`NRESET`. Параметры SPI: режим 0, частота по умолчанию 10 МГц (меняется командой
`SET_SPI_SPEED` из Java). Уровни — 3.3 В; при необходимости согласуйте.

**Какие пины ESP32-S3 нельзя использовать:** занятые встроенной флешкой/PSRAM
(обычно GPIO 26–32, а на модулях с octal PSRAM ещё 33–37) и strapping-пины
(0, 3, 45, 46 — влияют на режим загрузки). Нативный USB — это GPIO 19, 20.
Относительно безопасные: 4–18, 21, 38–44, 47–48 (всегда сверяйтесь с даташитом
конкретного модуля). Если VPP включается низким уровнем — поменяйте логику в
`vpp9v_level()` в `ad3s_types.hpp`.

### Компиляция

```cmd
idf.py build
```

- В VS Code: `F1` → **ESP-IDF: Build Project**.
- При первой сборке скачаются внешние компоненты в `managed_components/`
  (TinyUSB, led_strip) — это нормально.
- Готовый бинарник: `build/ad3s_prog_esp32.bin`.

### Прошивка контроллера

1. Подключите плату к ПК USB-кабелем (с передачей данных).
2. Определите COM-порт: **Диспетчер устройств → Порты (COM и LPT)** — найдите
   «USB Serial Port (COMx)» (мост CP210x/CH340) или «USB JTAG/serial debug
   unit» (нативный USB-CDC у S3).
3. Прошейте, подставив свой порт:
   ```cmd
   idf.py -p COM3 flash
   ```
   В VS Code: `F1` → **ESP-IDF: Flash** (метод UART или USB).

> Большинство плат S3 прошиваются прямо по USB без кнопок. Если плата не входит
> в boot-режим автоматически — удерживайте **BOOT**, кратко нажмите **RST/EN**,
> отпустите **BOOT** и повторите `idf.py -p COMx flash`.

### Монитор и проверка

```cmd
idf.py -p COM3 monitor
```

- Всё сразу: `idf.py -p COM3 build flash monitor`.
- В VS Code: `F1` → **ESP-IDF: Monitor**; «всё сразу» — `F1` →
  **ESP-IDF: Build, Flash and Monitor** (или кнопки на нижней панели ESP-IDF).
- Выход из монитора: `Ctrl+]`.
- В логе видны сообщения с тегом `ad3s_prog_esp32` и `usbcdc`, инициализация
  SPI и USB. Отсутствие `panic`/ошибок = прошивка встала корректно.

После загрузки ESP32 представляется виртуальным COM-портом. Запустите скачанное
приложение **workstation4ad3s** — оно подключится к этому порту, и можно
читать/писать регистры микросхемы.

### Частые проблемы

| Симптом | Решение |
|---------|---------|
| Плата не определяется, нет COM-порта | Кабель без данных (замените); не установлен драйвер USB-UART (CP210x/CH340 — с сайта производителя; нативный USB-CDC ставится сам). Другой порт/кабель. |
| `idf.py` не найдена | Вы не в окружении ESP-IDF. Откройте «ESP-IDF 5.5.2 CMD» или терминал ESP-IDF в VS Code. |
| `driver/pulse_cnt.h: No such file or directory` | Компонент `esp_driver_pcnt` уже в `main/CMakeLists.txt`. Выполните `idf.py fullclean`, затем `idf.py build`. |
| После правки Kconfig/CMake не пересобирается | `idf.py fullclean && idf.py reconfigure && idf.py build`. |
| Постоянная перезагрузка / `panic` при старте | Обычно инициализация адресной ленты на неверном пине или конфликт пина с флешкой. Выберите режим GPIO и свободный пин. |
| Чип не отвечает (workstation4ad3s не получает данных) | Проверьте разводку SPI, питание и уровни чипа, что `NRESET` отпущен, и что выбран правильный вариант платы. |
| Слишком длинный путь / ошибки CMake | Перенесите проект ближе к корню диска, например `C:\projects\ad3s_prog_esp32`. |

### Используемая конфигурация

- **ESP-IDF**: v5.5.2 (оттестировано; подойдёт любая более свежая 5.5.x)
- **Таргет**: ESP32-S3
- **Каталог установки ESP-IDF** (по умолчанию для 5.5.2):
  `C:\Espressif\frameworks\esp-idf-v5.5.2` — путь меняется в зависимости от
  выбранной версии (`esp-idf-v5.5.3` и т.п.)
- **Проект**: `ad3s_prog_esp32` → бинарник `build/ad3s_prog_esp32.bin`

Если `IDF_PATH` не задан, его можно установить вручную:
```cmd
set IDF_PATH=C:\Espressif\frameworks\esp-idf-v5.5.2
call %IDF_PATH%\export.bat
```
после чего выполнять `idf.py ...` как описано выше.
